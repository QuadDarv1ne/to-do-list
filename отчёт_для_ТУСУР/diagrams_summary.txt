# Список диаграмм проекта

## Созданные диаграммы:

1. **ER-диаграмма** (`er_diagram.mmd`)
   - Структура базы данных с 46 сущностями
   - Отношения между сущностями
   - Типы данных и связи

2. **IDO-диаграмма** (`ido_diagram.mmd`)
   - Входы (Input): Пользовательские действия, интеграции, события
   - Данные (Data): База данных, бизнес-логика, службы
   - Выходы (Output): Интерфейсы, уведомления, интеграции

3. **Структура проекта** (`project_structure.mmd`)
   - Организация файлов и директорий
   - Распределение компонентов
   - Количество файлов в каждом модуле

4. **Архитектурная диаграмма** (`architecture_diagram.mmd`)
   - Многоуровневая архитектура приложения
   - Взаимодействие между уровнями
   - Внешние системы и инфраструктура

5. **ER-диаграмма для печати** (`er_diagram_printable.mmd`)
   - Визуальная диаграмма всех сущностей
   - Группировка по функциональным модулям
   - Основные связи между сущностями

6. **Простая текстовая ER-диаграмма** (`er_diagram_simple.txt`)
   - Текстовое представление всех сущностей
   - Связи между сущностями
   - Индексы и типы данных

## Используемые технологии:

- **Mermaid**: Для создания диаграмм
- **Symfony 6.4**: Фреймворк приложения
- **Doctrine ORM**: Для работы с сущностями
- **PHP 8.3**: Язык программирования

## Основные компоненты системы:

- 46 сущностей в базе данных
- 67 миграций для управления схемой
- 92 сервиса бизнес-логики
- 67 контроллеров
- Модули: задачи, CRM, финансы, уведомления, пользователи

## Назначение диаграмм:

Эти диаграммы могут использоваться для:
- Документации проекта
- Объяснения архитектуры команде
- Представления на защите практики
- Планирования дальнейшей разработки
- Технической документации
