# Описание базы данных системы управления задачами

## Общая информация

Система управления задачами разработана с использованием фреймворка `Symfony 6.4` и `Doctrine ORM`

База данных использует `SQLite` для локальной разработки и `PostgreSQL` для продакшн-среды.

## Технические характеристики

- **Фреймворк**: `Symfony 6.4`
- **ORM**: `Doctrine ORM 2.17`
- **СУБД**: `SQLite (dev)` / `PostgreSQL (prod)`
- **Язык**: `PHP 8.3`
- **Архитектура**: `MVC с Domain-Driven Design`

## Основные сущности системы

### 1. Пользователи (users)

Основная сущность для управления доступом и авторизацией.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- username (VARCHAR(180), UNIQUE)
- email (VARCHAR(180), UNIQUE)
- roles (TEXT) - JSON массив ролей
- password (VARCHAR(255))
- first_name, last_name (VARCHAR(100))
- phone (VARCHAR(20))
- position, department (VARCHAR(255))
- timezone (VARCHAR(50))
- locale (VARCHAR(10))
- is_active (BOOLEAN)
- last_login_at (DATETIME)
- created_at, updated_at (DATETIME)

**Индексы:**

- idx_users_email (email)
- idx_users_username (username)
- idx_users_active (is_active)

### 2. Задачи (tasks)

Центральная сущность системы для управления задачами.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- title (VARCHAR(255))
- description (TEXT)
- status (VARCHAR(50)) - pending, in_progress, completed, cancelled
- priority (INTEGER) - 1-5
- due_date (DATE)
- start_date (DATE)
- completed_at (DATETIME)
- estimated_hours (DECIMAL(5,2))
- actual_hours (DECIMAL(5,2))
- user_id (INTEGER, FOREIGN KEY)
- category_id (INTEGER, FOREIGN KEY)
- parent_id (INTEGER, FOREIGN KEY)
- created_at, updated_at (DATETIME)

**Индексы:**

- idx_tasks_user (user_id)
- idx_tasks_status (status)
- idx_tasks_priority (priority)
- idx_tasks_due_date (due_date)
- idx_tasks_category (category_id)

### 3. Категории задач (task_categories)

Система классификации задач.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- name (VARCHAR(255))
- color (VARCHAR(7))
- icon (VARCHAR(50))
- is_active (BOOLEAN)
- user_id (INTEGER, FOREIGN KEY)
- created_at (DATETIME)

### 4. Теги (tags)

Гибкая система тегов для задач.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- name (VARCHAR(100))
- color (VARCHAR(7))
- user_id (INTEGER, FOREIGN KEY)
- created_at (DATETIME)

**Связующая таблица task_tag:**

- task_id (INTEGER, FOREIGN KEY)
- tag_id (INTEGER, FOREIGN KEY)

### 5. Комментарии (comments)

Система обсуждения задач.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- content (TEXT)
- task_id (INTEGER, FOREIGN KEY)
- user_id (INTEGER, FOREIGN KEY)
- parent_id (INTEGER, FOREIGN KEY)
- is_pinned (BOOLEAN)
- created_at, updated_at (DATETIME)

### 6. Время выполнения (task_time_tracking)

Учет времени выполнения задач.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- task_id (INTEGER, FOREIGN KEY)
- user_id (INTEGER, FOREIGN KEY)
- start_time (DATETIME)
- end_time (DATETIME)
- duration (INTEGER) - в секундах
- description (TEXT)
- is_billable (BOOLEAN)
- created_at (DATETIME)

### 7. Повторяющиеся задачи (task_recurrences)

Система повторяющихся задач.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- task_id (INTEGER, FOREIGN KEY)
- user_id (INTEGER, FOREIGN KEY)
- frequency (VARCHAR(20)) - daily, weekly, monthly, yearly
- interval (INTEGER)
- days_of_week (TEXT) - JSON массив
- day_of_month (INTEGER)
- month (INTEGER)
- start_date (DATE)
- end_date (DATE)
- next_run_date (DATE)
- is_active (BOOLEAN)
- created_at (DATETIME)

### 8. Шаблоны задач (task_templates)

Система шаблонов для быстрого создания задач.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- name (VARCHAR(255))
- description (TEXT)
- category_id (INTEGER, FOREIGN KEY)
- priority (INTEGER)
- estimated_hours (DECIMAL(5,2))
- user_id (INTEGER, FOREIGN KEY)
- is_active (BOOLEAN)
- created_at, updated_at (DATETIME)

### 9. Уведомления (notifications)

Система уведомлений пользователей.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- user_id (INTEGER, FOREIGN KEY)
- type (VARCHAR(50)) - task_assigned, deadline_approaching, etc.
- title (VARCHAR(255))
- message (TEXT)
- entity_type (VARCHAR(50))
- entity_id (INTEGER)
- is_read (BOOLEAN)
- read_at (DATETIME)
- created_at (DATETIME)

### 10. CRM-сущности

#### Клиенты (clients)

**Поля:**

- id (INTEGER, PRIMARY KEY)
- company_name (VARCHAR(255))
- inn, kpp (VARCHAR)
- contact_person (VARCHAR(255))
- phone, email (VARCHAR)
- address (TEXT)
- segment (VARCHAR(50)) - retail, corporate, vip
- category (VARCHAR(50)) - new, active, inactive
- manager_id (INTEGER, FOREIGN KEY)
- notes (TEXT)
- created_at, updated_at (DATETIME)

#### Сделки (deals)

**Поля:**

- id (INTEGER, PRIMARY KEY)
- title (VARCHAR(255))
- client_id (INTEGER, FOREIGN KEY)
- manager_id (INTEGER, FOREIGN KEY)
- amount (DECIMAL(15,2))
- currency (VARCHAR(10))
- status (VARCHAR(50))
- probability (INTEGER)
- expected_close_date (DATE)
- actual_close_date (DATE)
- created_at, updated_at (DATETIME)

### 11. Бюджеты (budgets)

Система управления бюджетами.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- name (VARCHAR(255))
- total_amount (DECIMAL(10,2))
- spent_amount (DECIMAL(10,2))
- start_date, end_date (DATE)
- created_by (INTEGER, FOREIGN KEY)
- status (VARCHAR(50))
- currency (VARCHAR(10))
- created_at, updated_at (DATETIME)

### 12. Продукты (products)

Каталог продуктов и услуг.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- name (VARCHAR(255))
- sku (VARCHAR(100), UNIQUE)
- description (TEXT)
- category (VARCHAR(100))
- price, cost (DECIMAL(15,2))
- unit (VARCHAR(50))
- is_active (BOOLEAN)
- created_at, updated_at (DATETIME)

### 13. Документы (documents)

Система управления документами.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- title (VARCHAR(255))
- content (TEXT)
- document_type (VARCHAR(50))
- status (VARCHAR(50))
- created_by (INTEGER, FOREIGN KEY)
- created_at, updated_at (DATETIME)

### 14. Цели (goals)

Система постановки и отслеживания целей.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- title (VARCHAR(255))
- description (TEXT)
- user_id (INTEGER, FOREIGN KEY)
- status (VARCHAR(50))
- priority (INTEGER)
- start_date, end_date (DATE)
- progress (INTEGER)
- created_at, updated_at (DATETIME)

### 15. Привычки (habits)

Система отслеживания привычек.

**Поля:**

- id (INTEGER, PRIMARY KEY)
- name (VARCHAR(255))
- description (TEXT)
- user_id (INTEGER, FOREIGN KEY)
- frequency (VARCHAR(20))
- target_count (INTEGER)
- is_active (BOOLEAN)
- created_at (DATETIME)

## Связи между сущностями

### Основные отношения:

1. **Один ко многим:**
   - User → Tasks
   - User → Categories
   - User → Tags
   - User → Comments
   - Task → Subtasks
   - Task → Comments
   - Task → TimeTracking
   - Category → Tasks
   - Client → Deals

2. **Многие ко многим:**
   - Tasks ↔ Tags (через task_tag)
   - Tasks ↔ Dependencies (через task_dependencies)

3. **Самореференсы:**
   - Task.parent_id → Task.id
   - Comment.parent_id → Comment.id

## Индексы и оптимизация

### Критические индексы:

- По полям внешних ключей (user_id, task_id, category_id)
- По часто используемым фильтрам (status, priority, due_date)
- По полям поиска (email, username, title)
- Составные индексы для сложных запросов

### Стратегии оптимизации:

- Использование Soft Delete через фильтры Doctrine
- Кэширование метаданных и запросов
- Ленивая загрузка связанных сущностей
- Пагинация для больших наборов данных

## Безопасность

### Меры безопасности:

- Хеширование паролей (`bcrypt`)
- Управление ролями (`RBAC`)
- Защита от SQL-инъекций через `Doctrine ORM`
- Валидация данных на уровне сущностей
- `Soft Delete` для критических данных

## Миграции

Система использует `Doctrine Migrations` для управления схемой базы данных.

Все изменения структуры хранятся в отдельных файлах миграций в директории `migrations/`

## Резервное копирование

**Регулярные резервные копии базы данных создаются через cron-задания:**

- Ежедневно - полная копия
- Ежечасно - инкрементальные изменения

## Мониторинг производительности

- Логирование медленных запросов
- Мониторинг использования памяти
- Анализ планов выполнения запросов
- Отслеживание времени отклика `API`
