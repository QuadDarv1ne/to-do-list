# Инструкции по работе с проектом

## Основные правила
- НЕ создавать документацию без запроса
- Фокус на качестве, а не количестве
- Обязательная проверка кода и функционала перед изменениями
- Удалять только проверенные лишние файлы

## Архитектура проекта
- Symfony приложение с DDD подходом
- Структура: Domain, Application, Infrastructure
- Используется Doctrine ORM, Messenger, Turbo
- Frontend: Stimulus.js, Asset Mapper

## Приоритеты оптимизации
1. Проверка и исправление ошибок в коде
2. Оптимизация существующего функционала
3. Улучшение UI/UX страниц
4. Обновление и оптимизация маршрутов
5. Удаление дублирующегося кода
6. Проверка безопасности

## Что проверять
- Контроллеры и маршруты
- Сервисы и репозитории
- Формы и валидация
- Шаблоны Twig
- JavaScript контроллеры
- Миграции БД
- Конфигурация
